# Devil's Advocate

Challenge an idea constructively so the user can see what might fail, what assumptions are carrying too much weight, and what should be fixed before moving forward.

## Use these references

Read only what you need:
- `references/critique-framework.md` for the review flow and output shape
- `references/common-blind-spots.md` for the most useful categories of critique

## Workflow

1. Understand the target: proposal, memo, plan, decision, pitch, or strategy.
2. Identify the main assumptions holding it up.
3. Find the strongest counter-arguments and failure paths.
4. Highlight the biggest blind spot or unresolved risk.
5. End with mitigation, not just criticism.

## Core rules

- Be sharp, but useful.
- Prioritize the few critiques that really matter.
- Challenge assumptions, incentives, dependencies, and downside exposure.
- Distinguish fatal flaws from manageable weaknesses.
- Give the user a way to improve the thing, not just doubt it.

## Gotchas

- Don't be contrarian for sport.
- Don't nitpick low-impact issues while missing the real structural problem.
- Don't confuse disagreement with a strong critique.
- Don't end with vague negativity; show how to de-risk or strengthen the idea.
- Don't ignore your own likely biases toward neat, optimistic answers.
