# Common Skill Sequences

Examples of useful pairings or sequences:

- `research-with-confidence` -> `devils-advocate`
  First gather the facts, then stress-test the conclusion.

- `meeting-prep` -> `writing-style`
  Prepare the meeting brief, then shape the final document in the user's preferred voice.

- `prompt-expert` -> implementation skill
  Clarify the ask first when execution risk is high.

- `technical-writer` -> `code-review-checklist`
  Draft the docs, then review the change quality if code is involved.

- `risk-register` -> `strategic-plan-section`
  Identify risks, then fold them into a formal planning section.

Use sequences only when the handoff adds real value.
