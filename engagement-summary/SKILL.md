# Engagement Summary Writer

Transform stakeholder consultation data into structured, actionable findings.

## Report Structure

```
WHAT WE HEARD: [Project Name]
Stakeholder Engagement Summary

EXECUTIVE SUMMARY
- Engagement overview
- Key themes (3-5 bullets)
- Top recommendations heard

1. ENGAGEMENT OVERVIEW
   1.1 Objectives
   1.2 Methods Used
   1.3 Participation Summary

2. WHO WE HEARD FROM
   2.1 Demographics
   2.2 Geographic Distribution
   2.3 Stakeholder Categories

3. KEY THEMES
   [Theme 1]
   [Theme 2]
   [Theme 3]

4. DETAILED FINDINGS
   4.1 [Topic Area 1]
   4.2 [Topic Area 2]
   4.3 [Topic Area 3]

5. RECOMMENDATIONS HEARD
   5.1 Short-term Actions
   5.2 Medium-term Initiatives
   5.3 Long-term Considerations

6. HOW INPUT WILL BE USED

APPENDICES
A. Survey Questions
B. Meeting Summaries
C. Raw Data Tables
```

## Transit-Specific Questions

### Common Topics to Cover

| Topic | Key Questions |
|-------|--------------|
| **Service quality** | Frequency satisfaction, reliability, coverage |
| **Accessibility** | Physical access, information access, affordability |
| **Routes** | Missing connections, priority areas, network design |
| **Schedules** | Span of service, weekend/holiday needs |
| **Fares** | Price sensitivity, payment preferences, pass options |
| **Stops/amenities** | Shelters, benches, lighting, information |
| **Customer experience** | App, website, customer service, safety |
| **Future priorities** | Where to invest limited resources |

## Writing Guidelines

1. **Let data speak** - Lead with numbers, then interpret
2. **Balance perspectives** - Include minority views, not just majorities
3. **Use direct quotes** - Humanizes the data
4. **Distinguish fact from opinion** - "Participants felt..." vs. "Data shows..."
5. **Avoid leading language** - Report what was said, not what you wanted to hear
6. **Acknowledge limitations** - Self-selection bias, sample size

## Handling Conflicting Input

When stakeholders disagree:

```markdown
### Divergent Views: [Topic]

**Perspective A** (held by [%] of respondents):
> "[Representative quote]"
Rationale: [Why some feel this way]

**Perspective B** (held by [%] of respondents):
> "[Representative quote]"
Rationale: [Why others feel this way]

**Staff observation:** [Neutral analysis of trade-offs]
```

## Common Pitfalls

- Reporting only positive feedback
- Ignoring statistically insignificant but important qualitative input
- Over-generalizing from small samples
- Missing the "so what" - data without interpretation
- Forgetting to thank participants
