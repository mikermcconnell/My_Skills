# Engagement Summary Writer

Transform stakeholder consultation data into structured, actionable findings.

## Use these references

Read `references/engagement-report-templates.md` when you need report structure, demographic templates, theme-analysis templates, stakeholder-group breakdowns, or closing sections.

## Workflow

1. Identify the engagement type and available inputs.
   Common inputs include surveys, open comments, meeting notes, workshop notes, and stakeholder interviews.
2. Quantify what can be quantified.
   Count participants, methods, response rates, common choices, and repeated themes.
3. Synthesize qualitative input into themes.
   Distinguish major themes, minority views, and notable tensions.
4. Keep findings tied to the engagement evidence.
   Separate what participants said from staff interpretation.
5. Close the loop.
   Explain how the input will be used and what happens next.

## Writing guidelines

- Let the data speak; lead with numbers when they are meaningful.
- Balance majority and minority views.
- Use direct quotes selectively to humanize the findings.
- Distinguish fact from opinion.
- Avoid leading language.
- Acknowledge limitations such as self-selection bias or small sample size.

## Handling conflicting input

When groups disagree:
- name the disagreement clearly
- summarize each side fairly
- explain the trade-off neutrally
- avoid forcing false consensus

## Output expectation

Most outputs should include:
- engagement overview
- who participated
- key themes
- quantitative findings when available
- stakeholder-group differences when useful
- how the input will be used

## Gotchas

- Reporting only positive feedback
- Ignoring statistically insignificant but important qualitative input
- Over-generalizing from small samples
- Missing the "so what" - data without interpretation
- Forgetting to thank participants
