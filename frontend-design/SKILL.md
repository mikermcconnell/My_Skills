# Frontend Design

Use this skill as the compatibility router for broad UI requests.

## Routing

- Use `ux-architect` when the structure, layout, or task flow is the main problem.
- Use `ui-designer` when the structure is fine but the surface needs a stronger visual system.
- Use `frontend-developer` when the main task is implementation, responsiveness, accessibility, or interaction bugs.
- Use `whimsy-injector` when the UI works but feels generic or emotionally flat.
- Use `brand-guardian` when the product needs a more coherent house style across screens.
- Use `friendly-design-theme` when the user wants the warm Scheduler 4 / Transit On-Demand visual language specifically.

## Default Approach

1. Inspect the existing product first.
2. Decide whether the problem is structure, styling, implementation, personality, or theme.
3. Apply the smallest useful combination of skills instead of treating every UI task as a full redesign.
4. Preserve accessibility and responsive behavior regardless of style direction.
