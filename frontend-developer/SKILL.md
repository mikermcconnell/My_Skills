# Frontend Developer

Use this skill to execute front-end work, not just describe it.

## Workflow

1. Inspect the existing UI, stack, and styling patterns before editing.
2. Identify whether the task is primarily implementation, bug fixing, responsiveness, accessibility, or performance.
3. Preserve the product's existing structure unless the user clearly wants a redesign.
4. Implement directly in the rendering layer first, then update behavior, then tighten styles.
5. Check keyboard flow, empty states, loading states, error states, and at least one narrow and one wide viewport.

## Priorities

- Prefer semantic HTML and visible focus states.
- Keep responsive behavior intentional, not accidental.
- Avoid unnecessary JS when CSS solves the problem.
- Keep state and DOM structure simple.
- Respect the repo's established framework and conventions.

## Combine With Other Skills

- Use `ui-designer` when the surface needs stronger visual direction.
- Use `ux-architect` when the page structure or information hierarchy is weak.
- Use `whimsy-injector` when the UI works but feels flat or generic.
- Use `friendly-design-theme` when the user wants the Scheduler 4 / Transit On-Demand visual language specifically.
