# RFP Drafter for Municipal Procurement

Create practical procurement documents for consulting services using clear scope, fair evaluation, and realistic municipal process structure.

## Use these references

Read only what you need:
- `references/rfp-structure-and-templates.md` for standard document structure, scope templates, evaluation templates, and pricing forms
- `references/procurement-practice-notes.md` for municipal and transit-specific considerations, timelines, and common pitfalls

## Workflow

1. Identify the procurement type and why it fits.
2. Clarify the scope, deliverables, mandatory requirements, and evaluation approach.
3. Draft the document using the standard structure instead of inventing a format each time.
4. Keep scope, evaluation, and submission requirements internally consistent.
5. Surface timeline, fairness, and procurement-risk issues before finalizing.

## Core rules

- Make the scope specific enough to price and execute.
- Keep mandatory requirements truly mandatory.
- Ensure evaluation criteria match the actual priorities of the procurement.
- Distinguish clearly between requirements, rated criteria, and pricing.
- Draft like a real municipal procurement document, not a casual project brief.

## Gotchas

- Don't create vague scopes that invite non-comparable bids.
- Don't overweight methodology while pretending price matters equally if it doesn't.
- Don't include mandatory requirements you don't actually need.
- Don't make the timeline impossible for the market or the evaluation team.
- Don't let appendices contradict the main document.
