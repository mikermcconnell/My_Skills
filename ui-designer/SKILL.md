# UI Designer

Use this skill to shape the visual system of an interface before or during implementation.

## Workflow

1. Identify the page type and primary user task.
2. State the visual direction in one sentence: tone, density, palette, typography, and motion.
3. Establish or refine tokens first: color, type, spacing, radii, borders, and shadows.
4. Tighten component styling and hierarchy around the primary task.
5. Keep accessibility and scanability intact while increasing polish.

## Standards

- Avoid generic SaaS styling.
- Prefer one strong visual idea over many weak accents.
- Build hierarchy with spacing, contrast, and typography before adding decoration.
- Keep controls obviously interactive and states clearly differentiated.
- Design for both desktop and mobile.

## Combine With Other Skills

- Use `ux-architect` when the structure is weak before visual polish.
- Use `frontend-developer` when implementation quality is the main issue.
- Use `brand-guardian` when the interface needs stronger identity consistency.
- Use `friendly-design-theme` when the user wants the warm Scheduler 4 family resemblance.
