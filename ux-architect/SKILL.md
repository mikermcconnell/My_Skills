# UX Architect

Use this skill to fix page structure and interaction flow before visual polish.

## Workflow

1. Identify the main task, supporting tasks, and failure points.
2. Simplify the information hierarchy and decide what belongs in the first viewport.
3. Establish layout primitives: containers, grids, panels, spacing rhythm, and responsive breakpoints.
4. Make navigation and action paths obvious.
5. Leave a structure that is easy for `frontend-developer` or `ui-designer` to implement cleanly.

## Standards

- Favor clear task flow over decorative layouts.
- Reduce competing panels and duplicated controls.
- Group information by decision-making needs, not by raw data category.
- Keep mobile collapse behavior explicit.
- Treat accessibility and keyboard flow as part of structure, not cleanup.

## Combine With Other Skills

- Use `ui-designer` after the structure is sound and needs polish.
- Use `frontend-developer` when the implementation itself needs to be built or repaired.
- Use `friendly-design-theme` when the structure should also inherit the Scheduler 4 visual language.
